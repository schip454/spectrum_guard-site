# Spectrum Guard — сайт охранного агентства

Лид-ген сайт под органику (Яндекс + Google), нейровыдачу (AEO/GEO) и Яндекс.Директ.
Бренд — Spectrum Guard, ответвление маркетингового агентства. Домен: `spectrumguard.ru`.

Это репозиторий со **спецификацией и реализацией**. Сначала читаются `docs/*`, затем
код пишется строго по ним. Спеки — источник правды; если код расходится со спекой,
правится код (или, по согласованию, спека — но не молча).

## Что строим

- Статический сайт на **Astro** (SSG), без SSR. Деплой как статика на шаред-хостинг reg.ru.
- Геосетка под 8 городов × 5 услуг + хабы + блог под SEO/AEO.
- Конверсия без обязательного телефона: форма заявки, кнопка в Telegram, обратный звонок.
  Заявки уходят на e-mail и в Telegram-бота. На сайте показываем один корпоративный
  мобильный номер.

## Стек и ограничения

- Astro 4+, TypeScript, контент-коллекции, динамические роуты для геосетки.
- Ноль тяжёлого JS. Интерактив (формы, меню, лайтбокс) — минимальный, на островах Astro
  или ванильном JS. Цель — Core Web Vitals в зелёной зоне (LCP < 2.5s, INP < 200ms, CLS < 0.1).
- Картинки — `astro:assets`, отдаём `webp`/`avif`, ленивая загрузка ниже первого экрана.
- Хостинг — шаред (PHP/Apache), Node на проде нет. Сборка локально/в CI → деплой `dist/`.
- Никаких внешних аналитик до запуска, кроме Яндекс.Метрики и Google Analytics (добавляются
  через единый компонент, легко включить/выключить).

## Принципы разработки

- **DRY.** Геостраницы и услуги генерируются из данных (`src/data/cities.ts`,
  `src/data/services.ts`), а не копипастой. Один шаблон — много страниц.
- **Spec-driven, поблочно.** Каждый блок (компонент/тип страницы) делается, ревьюится,
  закрывается, и только потом следующий. Порядок — в `docs/architecture.md` → «Порядок сборки».
- **Без ИИ-штампов.** Запрещены: дефолтные icon-паки (Font Awesome / Material), generic
  glassmorphism, декоративные градиенты, «floating cards» с большими тенями, шаблонные
  шрифты Inter/Roboto/Open Sans как основные. Дизайн — строго по `docs/design-spec.md`.
- **Честность данных.** Не выдумывать кейсы, отзывы, цифры, лицензии. Где контента ещё нет —
  ставить помеченные плейсхолдеры (`<!-- TODO:CONTENT -->`), не фейк.

## Definition of Done (для любого блока)

1. Соответствует спеке (ссылка на раздел в PR/коммите).
2. Адаптив до мобильного, видимый фокус клавиатуры, `prefers-reduced-motion` уважается.
3. Семантический HTML, корректная Schema.org (валидируется в валидаторе Яндекса и Google
   Rich Results) где предусмотрено.
4. Нет дефолтных icon-паков и запрещённых дизайн-приёмов (см. выше).
5. Lighthouse: Performance/SEO/Best Practices/Accessibility ≥ 95 на шаблонной странице.
6. Тексты — по `content-map.md`, без «воды» и без выдуманных фактов.

## Разработка и деплой

Стек реализации: **Astro 5** (SSG), TypeScript, контент-коллекции, `@astrojs/sitemap`,
`sharp` для генерации OG/иконок. Node 18.17+/20.3+/22+.

```bash
npm install
npm run dev      # локальный сервер (localhost:4321)
npm run check    # проверка типов Astro/TS
npm run build    # сборка в dist/ (prebuild генерит OG/иконки через sharp)
npm run preview  # предпросмотр собранной статики
```

Что генерируется само: `sitemap-index.xml` (все гео/хабы/блог, без 404), OG-картинка и
иконки (`scripts/gen-assets.mjs`). В `public/` лежат `robots.txt`, `llms.txt`, `.htaccess`,
PHP-обработчик заявок и фавиконки — всё это попадает в `dist/` при сборке.

### Приём заявок (бэкенд)

Форма шлёт POST на `public/api/lead.php` (вариант B из `docs/architecture.md`) → дублирует
в e-mail и шлёт в Telegram-бота. Секреты — в `server/lead.config.php` (по образцу
`server/lead.config.sample.php`), он в `.gitignore` и загружается на хостинг вручную, вне
перезаписи `dist/`. Анти-спам: honeypot + метка времени + файловый rate-limit по IP.

### Деплой (reg.ru shared)

`.github/workflows/deploy.yml` собирает и заливает `dist/` по FTP при push в `main`.
Секреты репозитория: `FTP_SERVER`, `FTP_USERNAME`, `FTP_PASSWORD`, опц. `FTP_SERVER_DIR`.
`lead.config.php` исключён из синхронизации, чтобы не затирать серверный секрет.

### Флаги (`src/config.ts`)

`TRUST_BLOCK_ENABLED`, `REVIEWS_ENABLED`, `CASES_ENABLED` — включаются одним движением,
когда придёт контент. Аналитика (`ANALYTICS`) — ID Метрики/GA, по умолчанию выключены.
