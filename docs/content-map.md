# Карта контента

Переменные подстановки: `{nom}`, `{prep}`, `{gen}` — склонения города из `cities.ts`;
`{service}` — название услуги; `{serviceUrl}` — строчная форма для title.
Тексты — живые, конкретные, без «воды» и без выдуманных фактов. Где данных нет — `TODO:CONTENT`.

## URL-структура

| Тип | URL | Кол-во |
|-----|-----|--------|
| Главная | `/` | 1 |
| Хаб услуги | `/uslugi/{service}/` | 5 |
| Хаб города | `/{city}/` | 8 |
| Услуга × город | `/{city}/{service}/` | 40 |
| Блог | `/blog/`, `/blog/{slug}/` | 1 + N |
| О компании | `/o-kompanii/` | 1 |
| Контакты | `/kontakty/` | 1 |
| Отзывы | `/otzyvy/` | 1 |
| Цены | `/ceny/` | 1 |

Slug услуг: `lichnaya-ohrana`, `ohrana-pri-pereezde`, `zashchita-pri-ugrozah`,
`konflikt-s-sosedyami`, `ohrana-meropriyatiy`.
Slug городов: `ekaterinburg`, `moskva`, `sankt-peterburg`, `ufa`, `chelyabinsk`, `perm`,
`tyumen`, `novosibirsk`.

## Главная `/`

- Job страницы: за 5 секунд показать «кто мы и что решаем» + увести в нужное направление.
- **Hero (первый экран):** во весь экран — `МЫ ЗАБОТИМСЯ О ВАШЕЙ БЕЗОПАСНОСТИ`, фоном —
  тёмный реальный видео-кадр (не ИИ-ролик с оружием). Снизу — 5 интент-карточек:
  1. Личная охрана → `/uslugi/lichnaya-ohrana/`
  2. Переезд от сожителя → `/uslugi/ohrana-pri-pereezde/`
  3. Вам угрожают? → `/uslugi/zashchita-pri-ugrozah/`
  4. Шумные соседи → `/uslugi/konflikt-s-sosedyami/`
  5. Проведение мероприятий → `/uslugi/ohrana-meropriyatiy/`
- Ниже героя — индексируемый смысловой блок (для SEO/ИИ нельзя экран без текста):
  Answer-First о компании, 5 услуг кратко, география (8 городов), почему мы, CTA.
- title: `Spectrum Guard — охрана и личная безопасность в 8 городах России`
- description: один абзац о личной охране, защите при угрозах, охране переездов и мероприятий
  + перечисление городов + способ связи.
- Schema: `Organization` (+ `areaServed` все города).
- CTA: форма заявки + Telegram.

## Хаб услуги `/uslugi/{service}/`

- Интент: общий по услуге без города + точка входа во все города.
- H1: `{service}` (например «Личная охрана»).
- title: `{service} — Spectrum Guard` (+ при необходимости «в России / 8 городов»).
- Answer-First (40–60 слов): что это за услуга, кому, как заказать.
- Секции: что входит · как работаем (этапы) · кому подходит · цены-ориентир (или «по запросу»)
  · FAQ (FAQPage) · блок «выберите город» → ссылки на 8 гео-страниц этой услуги.
- Schema: `Service` (areaServed = все города) + `FAQPage` + `BreadcrumbList`.
- Перелинковка: → 8 гео-страниц услуги, → соседние услуги, → релевантные статьи блога.

## Хаб города `/{city}/`

- Интент: «охрана {город}» общий + витрина всех услуг в городе.
- H1: `Охрана и безопасность {prep}` (напр. «…в Екатеринбурге»).
- title: `Охрана {prep} — Spectrum Guard`.
- Answer-First: какие услуги доступны {prep}, как работаем в городе, как связаться.
- Секции: 5 услуг карточками (→ `/{city}/{service}/`) · локальный блок (для `hasLocalProof`
  — местные фото/кейсы; иначе — локальный контекст без фейка) · FAQ · CTA.
- Schema: `Service`/`LocalBusiness` с `areaServed = {nom}`, **без адреса** · `BreadcrumbList`.
- Перелинковка: → 5 услуг в городе, → соседние города (переключатель).

## Услуга × город `/{city}/{service}/` (главный объём, 40 шт.)

- Интент: коммерческий гео-запрос «{serviceUrl} {nom}» / «{serviceUrl} {prep}».
- H1: `{service} {prep}` (напр. «Личная охрана в Екатеринбурге»).
- title: `{service} {prep} — заказать | Spectrum Guard`.
- description: что за услуга {prep}, кому подходит, как заказать (с топонимом в естественном
  падеже, без переспама).
- **Answer-First** (40–60 слов): прямой ответ — «{service} {prep}: что это, в каких ситуациях,
  как заказать у Spectrum Guard».
- Секции (DRY-шаблон, данные из `services.ts`):
  что входит · как проходит работа (этапы) · кому подходит · сколько стоит / от чего зависит ·
  локальный блок (фото/кейсы при наличии) · FAQ (FAQPage) · CTA.
- Schema: `Service` (serviceType из `services.ts`, provider→Organization, areaServed={nom})
  + `FAQPage` + `BreadcrumbList`.
- Перелинковка: → другие услуги {prep}, → эта услуга в соседних городах, → 1–2 статьи блога.
- Контент **уникален по городам**: для `hasLocalProof` — реально локальный; для остальных —
  переписанный, не клон (разные примеры/формулировки), топоним естественно.

## Интент-кластеры по услугам (для текстов, блога и AEO)

- **Личная охрана:** коммерч. «личная охрана {город}», «телохранитель {город}», «нанять
  охрану для человека». Проблемные: «нужна ли личная охрана», «сколько стоит телохранитель».
- **Охрана при переезде:** «охрана при переезде {город}», «сопровождение при переезде».
  Проблемные: «переезд от сожителя что делать», «как безопасно съехать», «забрать вещи
  при конфликте». Рамка строго правовая: сопровождение и присутствие, не «разобраться с кем-то».
- **Защита при угрозах:** «защита при угрозах {город}», «охрана если угрожают». Проблемные:
  «что делать если угрожают», «угрожает бывший». Рамка: законная защита и сопровождение.
- **Конфликт с соседями:** «{город} помощь с шумными соседями», «охрана при конфликте с
  соседями». Проблемные: «что делать с шумными соседями», «куда обращаться». Рамка: фиксация,
  присутствие, правовое сопровождение — НЕ давление на третьих лиц.
- **Охрана мероприятий:** «охрана мероприятий {город}», «охрана на мероприятие заказать».
  Проблемные: «как организовать охрану мероприятия», «сколько охранников нужно».

## Блог `/blog/`

- Лента + статья. Контент-коллекция (`src/content/blog`), frontmatter: title, description,
  slug, author, date, cover, relatedService, relatedCity?
- Каждая статья: Answer-First в начале, структура под извлечение, FAQ при уместности,
  внутренняя ссылка на профильную услугу/гео-страницу. Schema `Article` с автором (E-E-A-T).
- Стартовые темы (по проблемным кластерам выше) — список в TODO, тексты от заказчика/копирайтера.

## Служебные

- `/o-kompanii/`: о Spectrum Guard, опыт, география, принципы. Здесь же — слот трастового
  блока (за флагом). Без выдуманных фактов и без упоминания «лицензия будет позже».
- `/kontakty/`: корпоративный мобильный, e-mail, Telegram, форма, способы связи, география.
  Schema `Organization`/`ContactPoint`.
- `/otzyvy/`: реальные отзывы (TODO:CONTENT), Schema `Review`/`AggregateRating` только по факту.
- `/ceny/`: ориентиры или «по запросу» + форма. Без фейковых цифр.

## Тон текста (по design-spec → «Письмо»)

Спокойно, по делу, без запугивания и без пафоса. Активный залог, ясные глаголы, sentence case.
CTA называет действие: «Оставить заявку», «Написать в Telegram», «Заказать звонок».
